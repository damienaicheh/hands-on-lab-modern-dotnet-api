# Workshop 

Initialize a new TypeSpec project with the following command:

```bash
tsp init
```

select `Generic REST API` give a name `DocumentAPI` and select `OpenAPI 3.1 document` and `C# server stubs`.

Update the tspconfig.yaml file to generate both yaml and json OpenAPI documents:

```yaml
options:
  "@typespec/openapi3":
    emitter-output-dir: "{output-dir}/schema"
    # ADD THE FOLLOWING LINES
    file-type:
      - yaml
      - json
```

In the backdround we run the compilation of the TypeSpec files and generate the OpenAPI document and C# server stubs:

```bash
tsp compile . --watch 
```

Define the models inside the `spec/models` folder and the endpoints inside the `spec/endpoints` folder.

models/health.tsp
models/documents.tsp
models/error.tsp

endpoints/health.tsp
endpoints/document.tsp

Wrap everything inside the main.tsp file.

Creation of the C# dotnet client using the OpenAPI document generated by TypeSpec:
````bash
# Create a new console application
dotnet new console -o DocumentAPIClient

# Add dependencies for Kiota
cd DocumentAPIClient
dotnet add package Microsoft.Kiota.Bundle

# Generate the C# client using Kiota
kiota generate -l CSharp -c DocumentAPIClient -n DocumentAPI.Client --exclude-backward-compatible -d ./spec/tsp-output/schema/openapi.yaml -o ./DocumentAPIClient 
```
